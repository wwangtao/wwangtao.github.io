网页与Web应用的区别

12-20 2人

几年前，只要会 Dreamweaver、会写 jQuery 插件的人就能去 BAT 应聘前端开发职位；而如今，如果你不知道 React、Angular、AMD、CommonJS、Gulp、ES6 这些东西，你都不好意思自称前端开发了。

几年前，前端开发工程师还被习惯性得称作『美工』『切图仔』，而且工资比后端开发还低了一大截；而如今，没有哪个公司不对前端开发工程师求贤若渴，无论是创业公司，还是像 BAT 这样的大公司，都是如此。

为什么会这样？是什么让前端开发突然从低级的『美工』进化成了华丽的『工程师』，这些年究竟发生了什么？

我们就不得不说到今天的话题了：网页（web pages）与 Web 应用（web application）的区别了。

Web 应用的来历

如果你是 1995 之前出生的，你可能见过或听说过，以前的软件是装在光盘（甚至软盘）里面的。你买来光盘，然后安装到你的电脑（PC）中。比如 Office 软件、游戏软件，都是这样的。

后来我们的网速提高了，所以你可以直接下载安装文件来安装了（而且大部分都是盗版，因为正版还是以光盘的形式售卖，比如仙剑奇侠传）

如果你的 PC 配置高，那么这些软件运行就很流畅，如果配置低，那么就会很卡。

直到 2004 年 Gmail 出现。

Gmail 是 Google 推出的一个『网页』，你可以在上面收邮件、整理邮件、发邮件，以及设置收信规则等。而在这之前，我们想要收发邮件必须安装一个邮件客户端，比如 Foxmail。而 Gmail 意味着，你不需要安装任何东西，直接打开一个网页，就可以完成以前只能在桌面软件中才能完成的事情。

后来，我们看到了更多能代替桌面软件（甚至游戏）的网页了，这些网页，就是 Web 应用。

等 Gmail 传到中国的时候，中国的程序员都惊呆了，『JS 还能做出这么牛 X 的东西呀？』『怎么做的呀？』后来，大家才知道 AJAX 。我们熟知的 QQ 邮箱多年之后才做到跟 Gmail 体验差不多。

网页与 Web 应用的区别

那么什么样的页面才能算作『Web 应用』呢？它和普通页面有什么区别？

一般来说，普通的『网页』提供『内容』给你『看』，比如新网网站。

而 Web 应用，是让你在上面『干活儿』，也就是『做事』的。主要有下面几个特点：

  1. 你通过这个网页向服务器发出操作指令

    例如，你点击『发送邮件』按钮，实际上是通过 AJAX 向服务器发出一个请求，服务器收到请求后，执行『发送邮件』这个动作。

    当然，随着 PC 越来越厉害，你不需要把每个指令都发送到服务器，有些功能直接在 PC 上就能完成，比如有些 Web 应用可以讲 Markdown 转换为 HTML，整个过程都可以在 PC 上实现；再比如有些 Web 应用可以直接利用 PC 的 GPU 来实现图片滤镜功能。

  2. 这个网页帮你完成特定的功能

    例如高德地图可以帮你定位一个地方、豆瓣FM可以让你听歌。

    由于 Web 应用能很好地实现这些功能，而且使用方便无需安装，所以很多人使用 Web 应用代替了桌面软件。比如网易云音乐代替了千千静听。

实际上网页和 Web 应用的界限并不是那么明确，有很多网站是介于两者之间的。

比如新浪微博。有些人只看不回复，那么新浪微博对他来说就是一个浏览新闻的普通页面；而有些人使用了新浪微博的很多功能，发微博、发照片、回复和评论等，那么这时新浪微博就是一个 Web 应用了。

如何打造一个 Web 应用

我们知道普通网页仅靠 CSS 和 jQuery 就能实现。那么 Web 应用主要用到什么技术呢？

  1. 很多 AJAX

    由于 Web 应用要不停地向服务器发送请求，所以它要用到很多 AJAX。仅仅使用 jQuery.ajax 只能勉强满足需求。

  2. 渲染 HTML

    用户进行一次操作后，Web 应用需要马上告诉用户操作的结果是成功还是失败，所以它需要局部更新页面内容，这就需要在浏览器里渲染 HTML 了。

    最简单的实现方法是：$container.append('<h1>'+result+'</h1>')，但是如果 HTML 内容很多，我们就需要使用 JS 模板引擎了。

如果你的 Web 应用变得更复杂，那么下面的技术可能有用：

  1. 数据绑定

    由于 Web 应用主要做的事情包扩

    • 将服务器的数据同步到页面上
    • 将用户输入的数据同步到服务器和页面上

  2. 所以数据同步是一个频度很高的操作。数据绑定能让这一些都无感知地达成。例如 Angular 的数据单向绑定和双向绑定，非常方便。

  3. 嵌套的模板渲染

    嵌套模板的好处是提高复用率、降低耦合性。

  4. 路由

    由于页面被不停地局部刷下，所以页面状态就很混乱，连简单的前进和后退都无法支持。

    路由能让『前进』『后退』功能变得有意义，而且可以让页面的每个状态都能被记录下来。

Web 应用对前端工程师的要求

显然，仅靠『美工』和『切图仔』的专业知识，已经无法制作出一个 Web 应用了。

  1. 网页调试技术

    仅靠『所见即所得』，你是无法定位 Web 应用到底出了什么问题的。你还需要知道如何使用网络代理（Fiddler/Charles），还有浏览器开发者工具里的元素审查器、网络查看器、JS调试器等。

  2. HTTP

    你需要知道你向服务器发出了什么指令，动作是 GET 还是 POST，内容格式是 URL Encoded 还是 JSON。

  3. 富应用框架

    无论是 MVC 框架 Backbone、MVVM 框架 Angular，还是基于 React 的类 FLUX 框架，他们搭建复杂 Web 应用的好帮手。

  4. 模块化

    一个 Web 应用，可能有十个子功能，你如何以模块化的方式组织这些功能？

  5. 组件化

    一个功能独立的模块，如果需要被复用，也行你应该将它变为一个封装优雅、文档健全、功能强大的组件。

  6. 性能优化

    现在你的 Web 应用已经需要加载 50 个 CSS 文件和 100 个 JS 文件，如果没有任何优化，用户可能需要 10 秒才能下载完这些内容，怎么办？
