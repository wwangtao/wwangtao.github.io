常用 Git 命令清单

2016-12-20 22:39 前端基础 admin 2人

常用 Git 命令清单

12-20 2人

作者： 阮一峰

日期： 2015年12月 9日

我每天使用 Git ，但是很多命令记不住。

一般来说，日常使用只要记住下图6个命令，就可以了。但是熟练使用，恐怕要记住60～100个命令。




下面是我整理的常用 Git 命令清单。几个专用名词的译名如下。
  • Workspace：工作区
  • Index / Stage：暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库

一、新建代码库 

# 在当前目录新建一个Git代码库
{content}nbsp;git init

# 新建一个目录，将其初始化为Git代码库
{content}nbsp;git init [project-name]

# 下载一个项目和它的整个代码历史
{content}nbsp;git clone [url]

二、配置

Git的设置文件为.gitconfig，它可以在用户主目录下（全局配置），也可以在项目目录下（项目配置）。 

# 显示当前的Git配置
{content}nbsp;git config --list

# 编辑Git配置文件
{content}nbsp;git config -e [--global]

# 设置提交代码时的用户信息
{content}nbsp;git config [--global] user.name "[name]"
{content}nbsp;git config [--global] user.email "[email address]"

三、增加/删除文件 

# 添加指定文件到暂存区
{content}nbsp;git add [file1] [file2] ...

# 添加指定目录到暂存区，包括子目录
{content}nbsp;git add [dir]

# 添加当前目录的所有文件到暂存区
{content}nbsp;git add .

# 添加每个变化前，都会要求确认
# 对于同一个文件的多处变化，可以实现分次提交
{content}nbsp;git add -p

# 删除工作区文件，并且将这次删除放入暂存区
{content}nbsp;git rm [file1] [file2] ...

# 停止追踪指定文件，但该文件会保留在工作区
{content}nbsp;git rm --cached [file]

# 改名文件，并且将这个改名放入暂存区
{content}nbsp;git mv [file-original] [file-renamed]

四、代码提交 

# 提交暂存区到仓库区
{content}nbsp;git commit -m [message]

# 提交暂存区的指定文件到仓库区
{content}nbsp;git commit [file1] [file2] ... -m [message]

# 提交工作区自上次commit之后的变化，直接到仓库区
{content}nbsp;git commit -a

# 提交时显示所有diff信息
{content}nbsp;git commit -v

# 使用一次新的commit，替代上一次提交
# 如果代码没有任何新变化，则用来改写上一次commit的提交信息
{content}nbsp;git commit --amend -m [message]

# 重做上一次commit，并包括指定文件的新变化
{content}nbsp;git commit --amend [file1] [file2] ...

五、分支 

# 列出所有本地分支
{content}nbsp;git branch

# 列出所有远程分支
{content}nbsp;git branch -r

# 列出所有本地分支和远程分支
{content}nbsp;git branch -a

# 新建一个分支，但依然停留在当前分支
{content}nbsp;git branch [branch-name]

# 新建一个分支，并切换到该分支
{content}nbsp;git checkout -b [branch]

# 新建一个分支，指向指定commit
{content}nbsp;git branch [branch] [commit]

# 新建一个分支，与指定的远程分支建立追踪关系
{content}nbsp;git branch --track [branch] [remote-branch]

# 切换到指定分支，并更新工作区
{content}nbsp;git checkout [branch-name]

# 切换到上一个分支
{content}nbsp;git checkout -

# 建立追踪关系，在现有分支与指定的远程分支之间
{content}nbsp;git branch --set-upstream [branch] [remote-branch]

# 合并指定分支到当前分支
{content}nbsp;git merge [branch]

# 选择一个commit，合并进当前分支
{content}nbsp;git cherry-pick [commit]

# 删除分支
{content}nbsp;git branch -d [branch-name]

# 删除远程分支
{content}nbsp;git push origin --delete [branch-name]
{content}nbsp;git branch -dr [remote/branch]

六、标签 

# 列出所有tag
{content}nbsp;git tag

# 新建一个tag在当前commit
{content}nbsp;git tag [tag]

# 新建一个tag在指定commit
{content}nbsp;git tag [tag] [commit]

# 删除本地tag
{content}nbsp;git tag -d [tag]

# 删除远程tag
{content}nbsp;git push origin :refs/tags/[tagName]

# 查看tag信息
{content}nbsp;git show [tag]

# 提交指定tag
{content}nbsp;git push [remote] [tag]

# 提交所有tag
{content}nbsp;git push [remote] --tags

# 新建一个分支，指向某个tag
{content}nbsp;git checkout -b [branch] [tag]

七、查看信息 

# 显示有变更的文件
{content}nbsp;git status

# 显示当前分支的版本历史
{content}nbsp;git log

# 显示commit历史，以及每次commit发生变更的文件
{content}nbsp;git log --stat

# 搜索提交历史，根据关键词
{content}nbsp;git log -S [keyword]

# 显示某个commit之后的所有变动，每个commit占据一行
{content}nbsp;git log [tag] HEAD --pretty=format:%s

# 显示某个commit之后的所有变动，其"提交说明"必须符合搜索条件
{content}nbsp;git log [tag] HEAD --grep feature

# 显示某个文件的版本历史，包括文件改名
{content}nbsp;git log --follow [file]
{content}nbsp;git whatchanged [file]

# 显示指定文件相关的每一次diff
{content}nbsp;git log -p [file]

# 显示过去5次提交
{content}nbsp;git log -5 --pretty --oneline

# 显示所有提交过的用户，按提交次数排序
{content}nbsp;git shortlog -sn

# 显示指定文件是什么人在什么时间修改过
{content}nbsp;git blame [file]

# 显示暂存区和工作区的差异
{content}nbsp;git diff

# 显示暂存区和上一个commit的差异
{content}nbsp;git diff --cached [file]

# 显示工作区与当前分支最新commit之间的差异
{content}nbsp;git diff HEAD

# 显示两次提交之间的差异
{content}nbsp;git diff [first-branch]...[second-branch]

# 显示今天你写了多少行代码
{content}nbsp;git diff --shortstat "@{0 day ago}"

# 显示某次提交的元数据和内容变化
{content}nbsp;git show [commit]

# 显示某次提交发生变化的文件
{content}nbsp;git show --name-only [commit]

# 显示某次提交时，某个文件的内容
{content}nbsp;git show [commit]:[filename]

# 显示当前分支的最近几次提交
{content}nbsp;git reflog

八、远程同步 

# 下载远程仓库的所有变动
{content}nbsp;git fetch [remote]

# 显示所有远程仓库
{content}nbsp;git remote -v

# 显示某个远程仓库的信息
{content}nbsp;git remote show [remote]

# 增加一个新的远程仓库，并命名
{content}nbsp;git remote add [shortname] [url]

# 取回远程仓库的变化，并与本地分支合并
{content}nbsp;git pull [remote] [branch]

# 上传本地指定分支到远程仓库
{content}nbsp;git push [remote] [branch]

# 强行推送当前分支到远程仓库，即使有冲突
{content}nbsp;git push [remote] --force

# 推送所有分支到远程仓库
{content}nbsp;git push [remote] --all

九、撤销 

# 恢复暂存区的指定文件到工作区
{content}nbsp;git checkout [file]

# 恢复某个commit的指定文件到暂存区和工作区
{content}nbsp;git checkout [commit] [file]

# 恢复暂存区的所有文件到工作区
{content}nbsp;git checkout .

# 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变
{content}nbsp;git reset [file]

# 重置暂存区与工作区，与上一次commit保持一致
{content}nbsp;git reset --hard

# 重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变
{content}nbsp;git reset [commit]

# 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致
{content}nbsp;git reset --hard [commit]

# 重置当前HEAD为指定commit，但保持暂存区和工作区不变
{content}nbsp;git reset --keep [commit]

# 新建一个commit，用来撤销指定commit
# 后者的所有变化都将被前者抵消，并且应用到当前分支
{content}nbsp;git revert [commit]

# 暂时将未提交的变化移除，稍后再移入
{content}nbsp;git stash
{content}nbsp;git stash pop

十、其他 

# 生成一个可供发布的压缩包
{content}nbsp;git archive
